# test_todos

## 用途
待办事项 CRUD 端点的 API 测试 — 全面覆盖创建、列表、详情、更新、切换和删除操作。

## 关键文件
| 文件 | 说明 |
|------|------|
| `test_create.py` | 待办事项创建 — 必填字段、可选字段、带标签/分类、子任务 |
| `test_list.py` | 待办事项列表 — 分页、按完成状态/搜索筛选、排序 |
| `test_detail.py` | 待办事项详情 — 按 ID 获取、不存在返回 404、其他用户的待办事项返回 forbidden |
| `test_update.py` | 待办事项更新 — 部分更新、清除可选字段、标签替换、自引用父任务拒绝 |
| `test_toggle.py` | 待办事项切换 — 切换完成状态、幂等性 |
| `test_delete.py` | 待办事项删除 — 级联删除子任务、不存在返回 404 |

## AI 代理使用说明

### 在此目录工作
- 每个文件专注于一个 CRUD 操作，包含正向和反向测试用例
- 测试通过认证客户端使用 API 调用创建自己的数据
- 验证响应结构、状态码和数据完整性
- 使用 `utils/assertions.py` 中的断言进行一致的验证

### 常见模式
- 每个测试文件使用 `auth_client` fixture
- 测试数据在 setup 或 fixtures 中创建，在 teardown 中清理
- 子任务测试验证父子关系和级联行为
